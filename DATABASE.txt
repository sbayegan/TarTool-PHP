The database schema was created to help us keep track of the following information:
1- The information of users (e.g name, username, password)
2- Users votes
3- Users favourite objects.

Here are the attributes and datatypes of each table:

*USERS*
USERID INT NOT NULL, // PRIMARY KEY
USERNAME VARCHAR(20) NOT NULL,
PASSWORD VARCHAR(20), // CHANGE THIS TO 25 <<<<<<<<<<<<<<<<<<<<
NAME VARCHAR(20),
EMAIL VARCHAR(30),
COMPANY VARCHAR(30),
DATE DATE,
TYPE VARCHAR(10),
CONFIRMED BOOL DEFAULT FALSE

*RESOURCES*
RESOURCEID	INTEGER		// PRIMARY-KEY
TITLE		VARCHAR(200)
DESCRIPTION	VARCHAR(300)
MEDIUM		VARCHAR(5)
URL		VARCHAR(400)

// This table is used to keep track of the users favourited resources
*FAVOURITES*
USERID		INTEGER
RESOURCEID	INTEGER
DATE		DATE

// The table is used to keep track of votes
*VOTES*
USERID		INTEGER
RESOURCEID	INTEGER
DATE		DATE

*INTERESTS*
USERID		INTEGER
INTEREST	VARCHAR(10)

*CATEGORIES*
RESOURCEID	INTEGER
CAT 	VARCHAR(10)
SUB		VARCHAR(10)

*LAST*
ONCE  int
USER  int
RESOURCE  int
